Cwm How to make a release

Dealing with the testing suite

Make a test for your particular bug

The test case should be something.n3, where something is replaced by something more descriptive. Put it in the directory in testing where is most belongs.

  1. Edit test/regression.n3, list/detailed.tests online.tests math/detailed.tests norm/detailed.tests or cwm/detailed.tests, whichever the test best belongs in, putting the test in the same general area of the things in its catagory. Chose a refererence file (something-ref.n3)
  2. touch the reference file
  3. run make in the test directory --- your new regression test will fail
  4. It gives you a cp command to run to make the test succeed. Run that command.

Run make test

$ cd test

$ make

Insure that everything passes. If not, lather, rinse, repeat.

Sync with CVS

Add all the necesarry files

you added your testing files, as well as possibly adding other files. The command to run is:

$ cvs add <file or directory name>

note that add, unlike every other cvs command, is not recursive, so you need to add every directory and file individually

Commit your changes

the command is

$ cvs commit

if not given a file, will do the entire current directory. If it complains about a conflict, deal with the confict.

make a release

Insure you know what you are releasing

cd swap

cvs -q -n update # are you in sync? Check for C and ? in column 1 C means conflicy ? means a file CVS doesn't know about

cvs update

All of the usual changes needed to release

  1. Update changes.html
  2. If new release #, then cvs tag rel-x-y-z
  3. Make a message.txt file (see Makefil;e) for cvs checkin message
  4. run the command

    $ make release

  5. Mail public-cwm-announce
  6. Mail to public-cwm-bugs with "[closed] " in front of the subject line "Re:" explaining the bug fixed, point to new test
  7. in swap/admin, make to get new bug list (check with danc if doesn't work)
